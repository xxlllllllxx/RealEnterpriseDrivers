通过蓝牙® 传输文件

与其他蓝牙设备共享计算机上的本地文件和文件夹、在计算机和蓝牙设备之间传输任何类型的文件、甚至在蓝牙设备上创建或删除文件和文件夹，以及使用 Microsoft Office 菜单中的“发送向导”将文件发送到蓝牙设备。

支持的模式与角色：

  • 文件传输协议 (FTP)（客户端/服务器）
  • 基本成像配置文件 (BIP)（发起人/响应者）
  • 对象推送配置文件 (OPP)（客户端/服务器）

支持的功能：

  • 对象交换 (OBEX) 1.5 单响应模式 (SRM)
  • Microsoft Office 与对象推送模式 (OPP) 客户端的集成

与其他蓝牙设备共享本地文件夹

通过执行本用户指南文件夹共享设置部分中所述的以下步骤，可与其他蓝牙设备共享计算机上的文件夹。

共享文件夹用作通过 FTP 传输文件的根文件夹，以及通过 OPP 和 BIP 接收文件的默认文件夹。

浏览远程蓝牙设备的共享文件夹

连接至支持 FTP 的蓝牙设备后，一个类似 Explorer 的标准窗口可用于在设备和计算机之间进行正常文件操作，正如在本地执行操作一样：

  • 浏览文件和文件夹
  • 传输文件和文件夹
  • 删除文件和文件夹
  • 重命名文件和文件夹
    注意：如果远程设备支持低于 v1.2 版本的 FTP 服务器角色，则仅空文件夹可以重命名。
  • 创建文件夹

要从“服务”窗口访问远程设备中的共享文件夹：

  1. 打开设备上的“服务”窗口（请参阅添加并连接到新蓝牙® 设备）。
  2. 单击浏览远程文件夹。

要从 Windows 资源管理器访问远程设备中的共享文件夹：

  1. 打开计算机文件夹。
  2. 单击带远程设备名称的图标。

注意：如果远程设备暂时支持 FTP 服务器角色（例如，可以打开或关闭 FTP 服务器角色），则计算机文件夹中的链接不存在。在此情况下，使用远程设备“服务”窗口。

注意：在数据交换期间，远程设备可能需要密码，在这种情况下，两个设备必须输入相同的密码才可启用连接。如果密码不匹配，将终止此连接。

将文件发送到蓝牙设备

将文件发送到蓝牙设备向导用于将文件发送到远程蓝牙设备。向导能够发送单个文件，但可以同时启动多个向导。系统提供一系列步骤来选择目标远程设备和要发送的文件。将文件发送到蓝牙设备向导可在多个场景中执行。

注意：通常，BIP 用于将图片从移动设备发送到计算机。OPP 用于发送所有其他类型的文件。将文件发送到蓝牙设备向导支持这两种配置文件。

将文件从 Microsoft Office 发送到蓝牙设备

支持以下 Microsoft Office 版本：

  • Microsoft Office 2010
  • Microsoft Office 2013

支持以下 Microsoft Office 应用程序和项目：

  • Word：文档
  • Excel：工作簿
  • PowerPoint：演示文稿
  • Access：数据库
  • Outlook：约会、联系人、消息、注释、任务

Microsoft Office 项目在发送至远程蓝牙设备之前会自动保存。

注：如果 Microsoft Office 项目从不保存，将在 Word、Excel 和 PowerPoint 中显示保存提示。

注：如果 Microsoft Office 项目为数据库且刚刚创建，将重新启动访问应用程序。

要将 Microsoft Office 项目发送至远程蓝牙设备：

  1. 根据您使用的 Microsoft Office 版本和 Microsoft Office 应用程序，在共享、保存并发送或保存并发布子菜单中选择发送至蓝牙操作。请查阅 Microsoft Office 帮助，了解有关操作方法的指示。将文件发送到蓝牙设备向导会出现。
  2. 将文件发送到蓝牙设备向导打开之后，选择蓝牙设备以为其发送文件并单击下一步按钮。
  3. 然后，将出现文件传输进度窗口，项目将被发送到选定的蓝牙设备。传输完成后会通知您。

将文件从 Windows 资源管理器发送到蓝牙设备

要将文件从 Windows 资源管理器发送到蓝牙设备：

  1. 在 Windows 资源管理器中，右键单击文件图标并在上下文菜单的发送到项中选择蓝牙操作。将文件发送到蓝牙设备向导会出现。
  2. 将文件发送到蓝牙设备向导打开之后，选择蓝牙设备以为其发送文件并单击下一步按钮。
  3. 此时会出现文件传输进度窗口，选定的文件会被发送到选定的蓝牙设备。传输完成后会通知您。

将文件从服务窗口发送到蓝牙设备

要将文件从服务窗口发送到蓝牙设备：

  1. 单击服务窗口中的发送文件链接。此时将出现“打开”对话框窗口。
  2. 从列表中选择文件并单击打开按钮。将文件发送到蓝牙设备向导会出现。
  3. 将文件发送到蓝牙设备向导打开之后，单击下一步按钮。
  4. 此时会出现文件传输进度窗口，选定的文件会被发送到选定的蓝牙设备。传输完成后会通知您。

接收来自蓝牙设备的文件

其他蓝牙设备使用 BIP 或 OPP 启动到计算机的文件传输时，正在接收文件窗口将出现，为传入文件提供了默认目标路径。

注意：通常，BIP 用于将图片从移动设备发送到计算机。OPP 用于发送所有其他类型的文件。

在“接收文件”窗口中，可以更改保存传入文件/图片的默认目标文件夹。 为此，请按照以下步骤操作：

  1. 单击浏览并选择所需的文件夹窗口。
  2. 选择路径之后，单击接受开始文件传送。
  3. 要拒绝传入文件，请单击拒绝。

修改蓝牙设备中的文件和文件夹

支持 FTP 配置文件的蓝牙设备可更改本地共享文件夹中的文件和文件夹。如果远程设备尝试覆盖、重命名或删除共享内容，则需要特殊权限。先会出现权限请求窗口，然后才能执行上述更改。在权限请求窗口中有以下几个选项：

  • 选择只允许这一次并单击确定，以暂时允许从蓝牙设备作出的更改。
  • 选择总是允许此设备并单击确定，以允许在蓝牙设备配对后从蓝牙设备作出的更改。
  • 要暂时拒绝从蓝牙设备作出的更改，请单击取消。

如果要传输的文件未更改任何现有文件，则该文件将被传输，而不会出现权限请求窗口。

接收文件进度

其他蓝牙设备启动到计算机的文件传输且传输被接受时，正在接收窗口会出现：

  • 如果远程设备为 BIP 或 OPP 客户端，则正在接收窗口会出现。
  • 如果远程设备为 FTP 客户端，则可按照此用户指南中的文件夹共享设置部分所述的步骤配置显示的正在接收窗口。

正在接收窗口显示每个文件的以下信息：

  • 传输状态（已成功接收、未接收、正在传输等）
  • 当前传输文件的传输波特率
  • 成功接收的文件大小
